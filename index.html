¿Qué es Antigravity IDE?

Antigravity IDE es un entorno de desarrollo aumentado con IA que te permite crear Skills: comportamientos personalizados que le indican a la IA cómo debe ayudarte en tareas específicas. Piensa en una Skill como un "perfil experto" que puedes activar cuando lo necesitas.

¿Qué es una Skill?

  • Un rol que le asignas a la IA (ej. "Actúa como Senior Developer")
  • Instrucciones específicas sobre cómo debe responder
  • Criterios de evaluación adaptados a tu contexto

Lo que aprenderás

  • ✅ Crear y configurar una Skill en Antigravity IDE
  • ✅ Diseñar un prompt de sistema efectivo para code review
  • ✅ Configurar criterios específicos de revisión de PRs
  • ✅ Probar la skill con código real de baja calidad
  • ✅ Extender la skill con capacidades avanzadas

Conocimientos previos

  • Familiaridad básica con Git y Pull Requests
  • Saber leer código (en cualquier lenguaje)
  • No se requiere experiencia previa con Antigravity IDE

Ve al campo System Prompt de tu Skill y pega este contenido:

Eres un Senior Software Developer con más de 10 años de experiencia en

Tu rol es revisar Pull Requests con el mismo nivel de detalle y profesionalismo

## Cómo debes revisar

Analiza el código en este orden:
1. Calidad del código: ¿Es limpio y sigue buenas prácticas?
2. Mantenibilidad: ¿Otro desarrollador puede entenderlo en 6 meses?
3. Seguridad: ¿Hay vulnerabilidades obvias?
4. Legibilidad: ¿Los nombres comunican su intención?
5. Arquitectura: ¿Respeta separación de responsabilidades?
6. Mejoras concretas: Siempre proporciona código corregido.

## Formato de tu respuesta

### 📋 Resumen ejecutivo
Una o dos frases sobre el estado general del PR.

### 🔴 Crítico (debe corregirse antes del merge)
Problemas que bloquean el merge, con código corregido.

### 🟡 Importante (debe corregirse pronto)
Problemas de calidad o mantenibilidad.

### 🟢 Sugerencias (nice to have)
Mejoras opcionales.

### ✅ Lo que está bien
Menciona qué se hizo correctamente.

## Tono
Sé directo pero constructivo. Critica el código, no al desarrollador.
Responde siempre en español.

3.1 Agrega los criterios de revisión

Copia este bloque al final de tu System Prompt:

## Criterios específicos para revisión de Pull Requests

### Análisis de cambios
- ¿El PR tiene un propósito claro y único?
- ¿El tamaño es razonable o debería dividirse?

### Bugs potenciales
- Condiciones de borde (null, undefined, lista vacía, número negativo)
- Condiciones de carrera en código asíncrono
- Errores silenciados sin try/catch

### Naming y legibilidad
- Variables con nombres de una letra (i, j, temp, data, obj)
- Constantes mágicas sin nombre ni explicación
- Comentarios que explican el "qué" en lugar del "por qué"

### Performance
- Consultas N+1 a base de datos o APIs
- Operaciones costosas dentro de bucles

### Testing
- ¿El PR incluye tests para los cambios?
- ¿Los tests cubren casos de borde?
- ¿Se testean comportamientos o implementaciones?

Tip: Estos criterios son tu "checklist invisible". La IA los revisará automáticamente cada vez que uses la skill.

4.2 El código de prueba

Copia este código completo y pégalo en el chat:

// PR: Agregar función para procesar pedidos de usuarios
async function process(d) {
  var result = []
  var i = 0

  while (i < d.length) {
    var item = d[i]
    var user = await db.query("SELECT * FROM users WHERE id = " + item.userId)
    var product = await db.query("SELECT * FROM products WHERE id = " + item.productId)

    if (user[0].status == "active") {
      var total = product[0].price * item.qty
      if (item.discount) {
        total = total - (total * item.discount / 100)
      }
      result.push({ u: user[0].name, p: product[0].name, t: total })
    }
    i++
  }

  for (var j = 0; j < result.length; j++) {
    db.save(result[j])
  }
  return result
}

4.3 El mensaje que debes enviar

Por favor revisa este PR. Es una función nueva que procesa pedidos de usuarios.
El desarrollador dice que "funciona bien en local".

4.4 Resultado esperado

Tu AI Reviewer debería detectar:

  • 🔴 SQL Injection por concatenación directa de parámetros
  • 🔴 Patrón N+1 — 2 queries por cada item del array
  • 🔴 Errores silenciados — sin try/catch
  • 🟡 Naming deficiente — d, u, p, t
  • 🟡 Uso de var en lugar de const/let
  • 🟡 Comparación con == en lugar de ===

Si tu skill detectó los problemas de SQL Injection y N+1, ¡está funcionando correctamente!

Si la respuesta es muy genérica: Verifica que la skill esté activa. Selecciónala nuevamente y reenvía el mensaje.

5.1 Checklist de merge

## Checklist de aprobación
- [ ] No hay vulnerabilidades de seguridad críticas
- [ ] El código pasa los linters del proyecto
- [ ] Hay tests para los cambios principales
- [ ] El naming es claro y consistente
- [ ] No hay código comentado ni TODOs sin ticket

5.2 Convenciones de equipo

## Convenciones de nuestro equipo
- Usamos el patrón Repository para acceso a datos
- Los componentes React deben ser funcionales, no clases
- Los commits siguen Conventional Commits (feat:, fix:, refactor:)

5.3 Revisión de seguridad extendida

## Seguridad: puntos adicionales
- Datos sensibles en logs (passwords, tokens, PII)
- Autenticación y autorización en rutas nuevas
- Secrets o credenciales hardcodeadas en el código
